Ubike ELT Pipeline Project (GCP)

此專案展示了完整的雲端數據工程流程：使用 Terraform 建立 GCP 基礎設施，透過 Airflow排程，抓取 Ubike 與氣象 API 數據至 GCS，並利用 dbt 在 BigQuery 內進行資料清理與星型模型建置。


This project showcases a complete Cloud Data Engineering pipeline. It uses Terraform for GCP infrastructure deployment, Airflow for orchestration, extracts Ubike & Weather APIs to GCS, and leverages dbt for modeling (Fact/Dim tables) within BigQuery.

數據流程圖

本專案建構了完整的雲原生 ELT 數據管線。核心流程與技術棧如下：

  • 數據採集與載入 (E/L): 從 API 採集數據，並透過 Apache Airflow 協調，將數據載入 GCS (Data Lake) 及 BigQuery (Data Warehouse)。
  • 數據轉換與建模 (T): 使用 dbt 在 BigQuery 內部進行數據建模，生成乾淨的 Dim/Fact 分析表。
  • 基礎設施管理 (IaC): 整個雲端環境（GCP 資源）皆由 Terraform 進行自動化部署與管理。

Step 1：Terraform (IaC)建置雲端資源

本步驟使用 Terraform (Infrastructure as Code) 於 Google Cloud Platform (GCP) 上自動化部署所有所需的雲端資源。包含：GCS 儲存桶 (Data Lake)、BigQuery 資料集 (Data Warehouse) 以及用於運行 Airflow 的 Compute Engine (VM) 實例。

此做法確保了整個數據平台的環境配置具有可複製性和版本控制。

  • GitHub 連結

Step 2：Airflow + Cosmos 流程協調

本步驟使用 Apache Airflow 進行週期性排程與流程協調。DAGs 負責從 API 採集數據並載入 GCS 和 BigQuery。

導入 Cosmos 來無縫整合 dbt 專案，確保 E/L 完成後自動觸發 T 步驟，實現強健的 ELT 自動化。

  • GitHub 連結

Step 3：dbt 數據建模與品質保證

數據在 BigQuery 中透過 dbt (data build tool) 進行轉換與建模。我們依循 Kimball 維度建模原則，建構 Star Schema。

此外，利用 dbt Tests 實施數據品質檢查，確保模型具備唯一性 (unique) 和非空值 (not null) 等規範。

  • GitHub 連結

Step 4：數據成果：Dim/Fact 分析資料表

最終成果為 BigQuery 中乾淨、可供分析的 Dim/Fact 資料表。這些模型已完成數據的結構化、基礎清洗（如空值處理）和格式統一。

透過 dbt 實施的數據測試 (dbt Tests)，確保了最終分析模型具備高度的準確性和一致性，可直接連接至 Power BI 或 Google Data Studio 等 BI 工具，為業務分析師提供高價值的數據資產。
